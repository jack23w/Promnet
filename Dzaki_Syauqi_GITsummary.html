VSC (Version Control System)

Version Control System, atau VCS, adalah sistem yang dapat membantu engineer dan developer untuk melacak modifikasi kode dalam software. Sistem ini menjadi prioritas seluruh tim software development agar para anggota tidak berbenturan dan bertanggung jawab atas setiap perubahan yang mereka lakukan.

Penjelasan GIT

    GIT adalah sebuah tools bagi para programmer dan developer yang berfungsi sebagai control system untuk menjalankan proyek pengembangan software. GIT adalah singkatan dari Group Inclusive Tour. Tujuan penggunaan GIT yakni untuk mengelola versi source code program dengan menentukan baris serta kode yang akan ditambahkan atau diganti. Untuk instalasi GIT anda bisa klik link biru ini GIT Install

    Penjelasan GITHUB

    GITHUB merupakan layanan cloud yang berguna untuk menyimpan dan mengelola sebuah project yang dinamakan repository (repo git). Cara kerja pada GitHub harus terkoneksi pada internet sehingga tidak perlu meng-install sebuah software ke dalam perangkat keras. Hal ini memberikan keringanan penyimpanan komputer yang kita gunakan karena file project tersimpan oleh cloud GitHub.

    Konsep kerja GitHub pada dasarnya sama dengan Git yaitu dapat menulis source code secara individu atau tim. User interface yang tersedia pada GitHub lebih menarik dan mudah dipahami oleh pengguna awal. Pekerjaan secara tim, pengguna juga bisa melihat siapa penulis kode dan tanggal berapa kode tersebut dibuat.

    Terdapat fitur lain pada GitHub yaitu kita dapat membaca berbagai blog dan feed yang dibuat oleh sesama pengguna. Hal ini dimanfaatkan oleh pengguna seluruh dunia untuk saling berbagi ide pemrograman dan berdiskusi dalam menyelesaikan masalah. Tentunya postingan yang ada pada GitHub berkaitan dengan pemrograman. Sehingga Github telah menjadi forum diskusi para programmer seperti halnya media sosial.

    Perbedaan GIT dan GITHUB

    Lalu apakah GIT dan GITHUB itu sama? Simpelnya perbedaan nya terletak pada GIT berfungsi untuk menyimpan source code di lokal di komputer GITHUB berfungsi sebagai WEB untuk menyimpan dalam internet dan bersifat open source. Kalian juga bisa mentransfer code atau source kalian dari local ke web GITHUB untuk portofolio ataupun untuk collab bersama dengan yang lainnya. Berikut gambaran terminologi dan juga Fungsi GIT dan GITHUB

    Perintah - Perintah di GIT

      Penjelasan Perintah Git

      1. git config

      Deskripsi:git config adalah perintah dalam Git yang digunakan untuk mengatur dan mengelola konfigurasi Git. Konfigurasi ini dapat berupa pengaturan global (untuk seluruh repositori pada sistem), konfigurasi pada level repositori (hanya berlaku untuk repositori tertentu), atau konfigurasi pada level proyek (berlaku untuk proyek tertentu dalam sebuah repositori).

      2. git init

      Deskripsi: Perintah ini digunakan untuk menambahkan file ke indeks (staging area) untuk persiapan pengiriman ke repositori Git.

      3. git add

      Deskripsi: git add adalah perintah dalam Git yang digunakan untuk menambahkan perubahan pada berkas ke area staging (indeks). Ketika Anda melakukan perubahan pada berkas dalam repositori Git, perubahan tersebut tidak secara otomatis dimasukkan ke dalam komit.

      4. git clone

      Deskripsi: : Perintah ini digunakan untuk mengklon repositori Git yang ada dari URL ke dalam direktori lokal Anda.

      5. git commit

      Deskripsi: Perintah ini digunakan untuk membuat komit dengan file-file yang telah diindeks sebelumnya. Pesan komit digunakan untuk memberikan deskripsi singkat tentang perubahan yang Anda lakukan.

      6. git status

      Deskripsi: Perintah ini digunakan untuk melihat status file-file dalam repositori, termasuk file-file yang telah diubah, ditambahkan, atau dihapus.

      7. git push

      Deskripsi: Perintah ini digunakan untuk mengirim perubahan yang Anda buat dalam repositori lokal Anda ke repositori jarak jauh.

      8. git checkout

      Deskripsi: Perintah ini digunakan untuk beralih ke cabang lain dalam repositori.

      9. git remote

      Deskripsi: Perintah ini digunakan untuk melihat daftar remote repositories yang terhubung ke repositori lokal Anda.

      10. git branch

      Deskripsi: git branch adalah perintah dalam Git yang digunakan untuk melihat daftar cabang (branch) yang ada dalam repositori Git Anda. Branch adalah percabangan independen dari riwayat perubahan dalam repositori yang memungkinkan Anda untuk bekerja pada fitur, perbaikan bug, atau percobaan lain tanpa memengaruhi riwayat utama repositori.

      11. git pull

      Deskripsi: Perintah ini digunakan untuk mengambil perubahan terbaru dari repositori jarak jauh (remote repository) ke repositori lokal Anda.

      GITHUB PAGES

      GITHUB PAGES adalah layanan hosting statis yang disediakan oleh GitHub. Ini memungkinkan Anda untuk membuat situs web atau halaman web statis gratis yang di-hosting langsung dari repositori GitHub Anda. GitHub Pages sangat populer di kalangan pengembang perangkat lunak dan pengguna GitHub untuk mempublikasikan dokumentasi, portofolio, blog, atau situs web pribadi mereka.
